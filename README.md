# 프로젝트 제안서

김승현

### 1. 하는일

현재 SOC IP 개발팀에서 조직문화(CA)를 담당하고 있으며, 이전에는 Mobile 향 SoC의 Camera IP의 Android 기반 HAL Layer 개발을 담당하였습니다.

### 2. 프로젝트 주제

매년 9월에 한해동안의 조직문화 만족도 설문조사를 진행하고 있습니다. 해당 데이터는 기밀 데이터기때문에 사전에 분석한 데이터를 기반으로 데이터를 생성하여 진행할 예정입니다.

#### Feature(1점~5점, 혹은 지시함수 분류)

- 근속연수 (지시함수로 Fresh/Junior/Senior 분류)
- 직급(CL1,CL2,CL3,CL4)
- 소속 그룹
- 최종학력 : 고졸,대졸, 석사,박사
- 동료 관계
- 상사 만족도
- 교육 기회 만족도
- 평가 만족도
- 회의 문화
- 부서만족도
- 복지 만족도
- C레벨 만족도

#### Target

전반적 회사 점수 만족도(만족/보통/불만족)

#### 목표

- Feature의 학습을 통한 회귀모델 생성. 이를 통한 가장 영향력이 강한 Feature 도출
- 예측도 향상을 위한 차원축소를 통한 비지도 학습.
- 또한 비지도학습 분류를 통한 팀원들의 집단 성격 해석
- 부서별 성격 분석
- 이상데이터 분석을 통한 개인 면담 요소 탐색

#### 4. 예상되는 문제점 & 질문

점수에 대한 신뢰도가 낮은 편이기때문에 회귀모델의 신뢰도가 높다고 볼수는 없음.

보안 등급에 의해서 개개인의 대한 분석이 어려울수 있음(가상 데이터의 한계점)

매년 시행하기때문에 누적된 데이터라고 볼수있으나, OPI라는 제도 특성상 Scaling의 방식에 따라 결과가 달라질것으로 보임.

즉, 많은 시행착오를 거쳐 최적의 모델을 찾는것이 관건일것으로 보임.

다중공선성 문제가 있을 가능성이 있음(상사 만족도, 공정한 평가)
